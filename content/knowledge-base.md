# Bachelor's Thesis — Detail

## Abstract (German original)

Durch die fortschreitende Digitalisierung der Arbeitswelt ist die Notwendigkeit einer effektiven Geschäftsprozessmodellierung unbestreitbar. Insbesondere die Verwendung von Business Process Model and Notation (BPMN) 2.0 hat sich als Standard in der Industrie etabliert. Geschäftsprozesse werden jedoch von den Mitarbeitern oft als monoton und wenig motivierend wahrgenommen. Diese Arbeit zeigt auf, wie diesem Problem durch die Integration von Gamifizierungselementen – speziell Punkten, Auszeichnungen, Herausforderungen und einer Minimap – in BPMN 2.0 Modelle entgegengewirkt werden kann. Dafür wurde eine benutzerfreundliche webbasierte Umgebung für die Modellierung und Ausführen von gamifizierten Geschäftsprozessen konzipiert und mit React.js und TypeScript implementiert. Nach der Implementierung erfolgte eine detaillierte Analyse der funktionalen und nichtfunktionalen Anforderungen und erste Nutzertests zur Bewertung der Effektivität der spielerischen Elemente wurden durchgeführt. Diese Nutzertests konnten erste Tendenzen aufzeigen, dass die Gamifizierungselemente zu einem erhöhten Engagement der Benutzer in den Prozessen beitragen können. Allerdings muss die langfristige Auswirkung der Elemente auf die Motivation noch durch umfangreichere Studien erforscht werden. Die Ergebnisse dieser Arbeit weisen darauf hin, dass die Integration von Gamifizierung in BPMN 2.0 Modelle ein vielversprechender Ansatz zur Steigerung der Benutzerbeteiligung und Motivation ist.

## Motivation & Goal

Traditional BPMN business process models are often perceived as monotonous and unmotivating by end users. The thesis explores how gamification mechanics can be embedded directly into BPMN 2.0 models — both at design time (editor) and at runtime (engine) — to increase user motivation without changing the underlying process semantics. Supervisor: Prof. Dr. Manfred Reichert (DBIS, Ulm University). Advisor: Michael Winter.

## BPMN Editor (portfolio file: bachelor)
Node-based drag-and-drop web editor built on React Flow. Supports standard BPMN 2.0 elements (Start, End, Activity, Gateway, Sequence Flow) plus custom gamification elements:
- **Activity** — supports three task types: text input (with optional regex validation), single-choice, multiple-choice. Each activity can store the user's answer in a named variable and trigger a gamification event on completion.
- **Challenge node** — a container spanning multiple activities with a configurable time limit; rewards the user if all tasks are completed within time; can have additional conditions on the reward.
- **GamificationEvent node** — awards XP, coins, or unlocks a badge when reached; badge conditions can be hidden so they're only revealed on completion (encouraging re-exploration).
- **Info node** — displays a message to the user mid-process.
- **Gateway** — routes the process based on variable conditions (e.g., user's previous answer).

Export: to BPMN 2.0 XML (compatible with other BPMN tools) and to internal React Flow JSON. Loading back: only internal JSON format (BPMN→JSON transformation for challenges was not completed due to time constraints — the one unmet requirement).
[Live Demo](https://gbpmneditor.mertendieckmann.de) · [GitHub](https://github.com/MertenD/gamificated-bpmn-editor)

## BPMN Engine (portfolio file: bachelor)
Browser-based runtime that imports and executes the gamified BPMN JSON models. Supports all 7 node types: Start, End, Activity, Gateway, Challenge, GamificationEvent, Info.

Execution loop: load process → start at Start node → render current node's UI → handle user input → evaluate exit conditions/gateway branches → advance to next node → repeat until End node.

Variable management: user inputs stored in named variables; XP, coins, and badges tracked at all times; after each activity the engine checks whether any gamification reward conditions are met.

State management via Zustand. [Live Demo](https://gbpmnengine.mertendieckmann.de) · [GitHub](https://github.com/MertenD/gamificated-bpmn-engine)

## 4 Gamification Elements

1. **Points (XP & Coins)** — XP builds up a level shown to the user (progress indicator, social comparison). Coins can be used as in-process currency (e.g., buying an extra attempt at a failed challenge). Both are awarded via GamificationEvent nodes with configurable amount and trigger conditions.

2. **Badges (Auszeichnungen)** — unique collectibles unlocked by reaching GamificationEvent nodes. The user always sees how many badges exist in a process and which are already unlocked. Key design: badge conditions can be hidden — the user only discovers what triggers a badge after unlocking it, incentivising repeated runs to find all badges.

3. **Challenges (Herausforderungen)** — timed tasks spanning one or more activities. The user must complete all activities within the time limit to earn the reward. Failing redirects the process (gateway-based). Adds urgency and flow-state focus.

4. **Minimap** — a visual map of the full process graph shown during execution. Displays the current position, already-visited nodes, upcoming paths, and where rewards are available. Inspired by Duolingo's skill tree and Detroit: Become Human's chapter overview. Shows unexplored branches, motivating users to replay to find all paths and rewards.

## Results & User Testing

User test scenario: a maths quiz with a timed challenge. Correct answers awarded XP; completing all questions within time awarded a badge. A second badge was hidden behind a non-obvious condition (answering all questions wrong). 

Key findings:
- All test participants repeated the quiz at least once to unlock the hidden badge — confirms badges drive re-engagement
- Coins without a purpose (no shop/trade mechanic in the test) did not motivate; recommendation: integrate coins as spendable currency within processes or a cross-process shop
- Hidden badge conditions can cause frustration if not discovered quickly; hint system recommended
- Minimap successfully showed unexplored paths and motivated users to explore them

Requirements met:
- Editor: 6 of 7 functional requirements fully met (FAED007 — loading BPMN XML back into editor — partially implemented due to complex Challenge transformation)
- Engine: 6 of 6 functional requirements fully met
- Non-functional: Usability largely met (Nielsen's 10 heuristics), Maintainability met (modular architecture), Portability met (browser-based, OS-independent; not designed for mobile)

## Conclusion (German original)

In dieser Arbeit wurde das Konzept der Gamifizierung von Geschäftsprozessen erörtert und darauf aufbauend eine webbasierte Umgebung für die Erstellung und Ausführung von gamifizierten BPMN 2.0 Modellen implementiert. Dabei wurden Punkte, Auszeichnungen, Herausforderungen und eine Minimap als motivierende Gamifizierungselemente integriert. Die Benutzerfreundlichkeit und Wirksamkeit dieser Elemente wurden anhand von Nutzertests evaluiert, welche sowohl positive Aspekte als auch Bereiche für Verbesserungen aufzeigten. Zusammenfassend hat diese Bachelorarbeit gezeigt, dass die Integration von Gamifizierungselementen in BPMN-Prozesse machbar und wirkungsvoll ist. Die Arbeit hat konkrete Methoden und Tools zur Implementierung dieser Elemente präsentiert und erste empirische Hinweise auf ihre Wirksamkeit geliefert.

Future work: user profiles, a shop system for coins, leaderboards for social competition, longer longitudinal studies on motivation effects.

Tech stack: React, TypeScript, React Flow, Zustand, Create React App, Docker.

# Master's Thesis — Full Abstract & Conclusion (German)

## Abstract

In der EU ist die Einhaltung der Datenschutz-Grundverordnung (DSGVO) in Geschäftsprozessen essenziell, da bei Nichteinhaltung hohe Bußgelder drohen können. Jedoch ist eine manuelle Prüfung von Prozessmodellen in der Praxis aufwendig und fehleranfällig. Diese Arbeit untersucht daher, wie Large Language Models (LLMs) DSGVO-kritische Aktivitäten in BPMN-Prozessmodellen automatisiert identifizieren können. Hierzu wird (1) eine Klassifizierungspipeline mit Zero-Shot-Prompting und strikt strukturiertem JSON-Output, ergänzt um id-Validierung/-Vervollständigung und einem automatischen Retry-Mechanismus, (2) ein Labeling-Tool für gelabelte BPMN-Testfälle sowie (3) ein Evaluationsframework mit deklarativer Konfiguration, standardisierter HTTP-Schnittstelle und Frontend für reproduzierbare, vergleichbare Experimente entwickelt.

In einer empirischen Studie werden 13 LLMs auf 25 Testfälle über fünf Wiederholungen evaluiert. Neun von dreizehn Modellen erreichen einen F1-Score von ≥ 0,80. Spitzenreiter sind Qwen3-235B-A22B-Thinking-2507 (F1=0,874, Recall=0,932), GPT-OSS-20B (F1=0,866, Recall=0,918) und DeepSeek-R1-Distill-Qwen-14B (F1=0,848). Unter den EU-Modellen stechen Mistral Medium 3.1 (F1=0,843) und Mistral-Large-Instruct-2411 (F1=0,823) heraus. GPT-4o erzielte die höchste Precision (0,892), verfehlte jedoch mit Recall=0,762 die Mindestanforderung.

Fehleranalysen zeigen vor allem False Positives bei fehlendem Kontext und False Negatives bei mehrstufigen Datenflüssen. Insgesamt eignen sich aktuelle LLMs gut für ein automatisiertes, Recall-orientiertes Vorscreening von BPMN-Prozessen, jedoch bleibt eine nachgelagerte fachliche Prüfung erforderlich.

## Conclusion

Diese Arbeit untersuchte, inwieweit moderne LLMs DSGVO-kritische Aktivitäten in BPMN-Prozessmodellen zuverlässig identifizieren können. In einer systematischen Evaluation mit 13 Modellen, 25 Testfällen und jeweils fünf Wiederholungen erreichten neun Modelle die Zielwerte (F1 ≥ 0,80). Offene Modelle wie Qwen3-235B und GPT-OSS-20B kombinierten hohen Recall mit solider Precision. Kleinere Modelle (≤ 25B) hielten mit größeren mit und sind für On-Premises-Szenarien attraktiv. Europäische Modelle (Mistral Medium 3.1, Mistral-Large-Instruct-2411) erwiesen sich als wettbewerbsfähig.

Die entwickelte Infrastruktur — Klassifizierungspipeline, Labeling-Tool und Evaluationsframework — schafft eine belastbare Basis für zukünftige Arbeiten und erlaubt neue Modelle, Prompting-Strategien und Testdaten nahtlos zu integrieren. Fazit: LLMs bilden in Kombination mit einer robusten Pipeline ein wirksames Screening-Werkzeug. Internationale Spitzenmodelle lieferten die besten Ergebnisse; europäische und kleinere Modelle sind unter Datenschutz- und On-Premises-Anforderungen konkurrenzfähig.
